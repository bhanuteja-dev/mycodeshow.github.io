Phase 5 — Distributed Systems Core

▼
No Pluralsight coverage for this phase. These are the topics that separate a backend engineer from a distributed systems engineer. Read Kleppmann before anything else here.
BookTheory foundation — read first
Designing Data-Intensive Applications — Kleppmann, chapters 5–9
The distributed systems bible. Ch5: replication (leader/follower, multi-leader, leaderless). Ch6: partitioning + secondary indexes. Ch7: transactions + isolation levels. Mandatory before studying Kafka or designing any data-intensive system.
